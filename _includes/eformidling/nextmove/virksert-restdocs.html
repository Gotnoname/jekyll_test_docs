Participants

Find a participant

A GET request is used to find a given participant

/api/v1/participants/{qualifiedIdentifier}
Parameter Description

qualifiedIdentifier

scheme + identifier = qualifiedIdentifier. Example 'iso6523-actorid-upis::0192:910539000'
Curl
$ curl 'http://localhost:8080/api/v1/participants/iso6523-actorid-upis::0192:910539000' -i -X GET \
    -H 'Accept: application/json'
Request
GET /api/v1/participants/iso6523-actorid-upis::0192:910539000 HTTP/1.1
Accept: application/json
Host: localhost:8080
Ok response
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 114

{
  "id" : 10,
  "identifier" : "0192:910539000",
  "scheme" : "iso6523-actorid-upis",
  "name" : "Grimsby"
}
Error response
HTTP/1.1 404 Not Found
Content-Type: application/json
Content-Length: 88

{
  "errors" : [ null, "no.difi.virksert.server.lang.ParticipantNotFoundException" ]
}

If the given participant does not exist then 404 Not found will be returned.

Add new participant

A POST request is used to add a new participant

/api/v1/participants/{qualifiedIdentifier}/{name}/
Parameter Description

qualifiedIdentifier

scheme + identifier = qualifiedIdentifier. Example 'iso6523-actorid-upis::9908:910539000'

name

Name of the participant
Curl
$ curl 'http://localhost:8080/api/v1/participants/iso6523-actorid-upis::0192:910539000/Grimsby/' -i -X POST \
    -H 'Content-Type: application/json' \
    -H 'Accept: application/json'
Request
POST /api/v1/participants/iso6523-actorid-upis::0192:910539000/Grimsby/ HTTP/1.1
Content-Type: application/json
Accept: application/json
Host: localhost:8080
Ok response
HTTP/1.1 201 Created
Error response1
HTTP/1.1 409 Conflict
Content-Type: application/json
Content-Length: 154

{
  "errors" : [ "Participant iso6523-actorid-upis::0192:810076402 already exists", "no.difi.virksert.server.lang.ParticipantAlreadyExistsException" ]
}
Error response2
HTTP/1.1 405 Method Not Allowed
Allow: GET,DELETE

If the participant already exists then 409 Conflict will be returned. If the request is missing a path parameter the method will return an 500 Error. Both name and identifier are required.

Find all certificates

A GET request is used to find and return a list of all certificates from any participant. Can return multiple certificates.

Request parameters are optional.

This endpoint can take one out of two possible request parameters, or be called without any parameter. This is used to find all certificates expiring within given amount of days or a before a given date. The request parameters are mutually exclusive. Examples shown below.

A POST request to add a new admin user, no specific participant need to be supplied. Default the values other than email and name may be empty. The API does not support adding users to a specific participant due to current use cases, this might be added in the future.

Request body
{
  "id" : 1337,
  "identifier" : "UserDummyIdentifier",
  "participant" : {
    "id" : 10,
